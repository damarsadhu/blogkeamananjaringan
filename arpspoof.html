1. Pendahuluan

ARP Spoofing (atau ARP Poisoning) adalah teknik serangan Man-in-the-Middle (MITM) yang digunakan pada jaringan lokal (LAN). Serangan ini memungkinkan penyerang untuk mengintersepsi lalu lintas data antara dua perangkat di jaringan, misalnya antara komputer korban dan router.

Tujuan dari tutorial ini adalah untuk memahami cara kerja serangan ini dari sudut pandang defensif, sehingga kita dapat mengetahui cara mencegahnya.

2. Langkah 1: Install dsniff

Alat arpspoof yang populer untuk demonstrasi serangan ini adalah bagian dari paket dsniff. Anda dapat menginstalnya di sistem berbasis Debian (seperti Kali Linux) dengan perintah:

sudo apt update sudo apt install dsniff

3. Langkah 2: Aktifkan IP Forwarding

Agar penyerang dapat meneruskan paket dari korban ke router (dan sebaliknya), mereka harus mengaktifkan IP forwarding di sistem operasinya. Jika tidak, serangan ini hanya akan memutus koneksi internet korban (menjadi serangan Denial of Service).

Perintah untuk mengaktifkan IP forwarding di Linux adalah:

sudo sysctl -w net.ipv4.ip_forward=1

Anda dapat memeriksa statusnya dengan cat /proc/sys/net/ipv4/ip_forward (hasilnya harus 1).

4. Langkah 3: Konsep Serangan

Serangan ini memerlukan dua proses yang berjalan bersamaan di terminal penyerang. Tujuannya adalah untuk "meracuni" cache ARP dari korban dan router.

Asumsi IP:

  • IP Korban: 192.168.1.100
  • IP Router/Gateway: 192.168.1.1
  • Interface Penyerang: eth0

Proses 1: Meracuni Korban

Penyerang mengirimkan paket ARP ke korban (192.168.1.100) secara terus-menerus. Paket ini memberi tahu korban bahwa alamat MAC untuk IP router (192.168.1.1) adalah alamat MAC milik penyerang.

Proses 2: Meracuni Router

Pada saat yang sama, penyerang mengirimkan paket ARP ke router (192.168.1.1). Paket ini memberi tahu router bahwa alamat MAC untuk IP korban (192.168.1.100) adalah alamat MAC milik penyerang.

Akibatnya, semua lalu lintas dari korban ke internet akan dikirim ke penyerang terlebih dahulu, dan semua lalu lintas dari internet ke korban juga akan melewati penyerang.

5. Peringatan Etika

Penting untuk diingat bahwa melakukan serangan ini di jaringan yang bukan milik Anda, atau tanpa izin tertulis yang jelas, adalah tindakan ilegal dan tidak etis. Teknik ini hanya boleh dipraktikkan di lingkungan laboratorium yang terisolasi (misalnya, menggunakan Virtual Machine) untuk tujuan pendidikan dan penelitian keamanan.

6. Cara Mitigasi (Pencegahan)

Memahami cara kerja serangan ini membantu kita untuk bertahan. Berikut adalah beberapa metode pencegahan:

  • Entri ARP Statis: Metode ini melibatkan pengaturan manual alamat MAC yang benar untuk IP tertentu (seperti router) di setiap komputer. Ini efektif tetapi sulit dikelola di jaringan besar.
  • Dynamic ARP Inspection (DAI): Ini adalah fitur keamanan pada switch jaringan yang canggih. DAI memeriksa setiap paket ARP dan membuang paket yang dianggap tidak sah atau palsu.
  • Alat Pemantauan: Perangkat lunak seperti arpwatch dapat memonitor jaringan untuk perubahan alamat MAC yang mencurigakan dan memberi tahu administrator.
